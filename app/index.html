Notes:

  • Currently the button above is set up to simply do transactions from the contract to account 1.
  • Check code comments for which addresses go in which location in the code. They are marked.
  • Contract addresses come from truffle deploy terminal.
  • Testing accounts come from the testrpc terminal.

  • To unlock accounts for testing, boot up the testrpc using the command: testrpc --secure -u 0 -u 1
    -u is unlock, the numbers are accounts, 0 and 1

  • The browserfy command is: browserify app.js -o bundle.js
    and must be done from within the javascript file in the WebApp part.

  • The python server for hosting the Dapp is: python3 -m http.server 8000

  • The Javascript API is important but always make sure you are using the 0.x.x one instead of the 1.0 one which isn't released yet.

  • To test transactions using .send or .transfer of ether directly from one account to the contract The DApp and contract must be connected to the test network through geth or something. Ropsten not Morden. Until then, the code will fail forever unless I can test that it works and move on.
